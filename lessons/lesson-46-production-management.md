## Learning Outcomes

By the end of this lesson, the learner should be able to:
1. Apply the **Scope Test** to evaluate whether an animation project is the right size before starting.
2. Create a production plan using the three-phase pipeline (pre-production, production, post-production) with realistic time estimates using the **2x Rule**.
3. Value the principle that **"done is better than perfect"** and demonstrate self-management by completing a scoped animation project.

## Key Inquiry Question

> How do we plan and finish an animation about **Kasa the Green Turtle (kasa)**
> swimming through Watamu Marine Park — without running out of time or
> giving up halfway?

## Local Context: Kenyan Ocean Animal

**Animal:** Green Sea Turtle — *Kasa* (Swahili)  
**Habitat:** Seagrass beds of Watamu Marine National Park, Diani reefs, and Kisite-Mpunguti Marine Reserve. Nests on beaches at Watamu and Diani.  
**Why this animal?** The green sea turtle has a clear, simple story to animate: it glides through the water, encounters a plastic bag (mistakes it for a jellyfish), and either eats it (sad ending) or avoids it (hopeful ending). This gives learners a **short, achievable project** (10-15 seconds) that has emotional weight, a conservation message, and enough complexity to require real production planning — perfect for teaching scoping and project management.  
**Conservation note:** Green sea turtles are **endangered** (IUCN Red List). Their main threats are plastic pollution (they mistake plastic bags for jellyfish), fishing bycatch, and habitat loss. KWS (Kenya Wildlife Service) and Local Ocean Conservation Trust in Watamu run nesting protection programs — they patrol beaches at night during nesting season, rescue trapped turtles, and rehabilitate injured ones. Students can support conservation by reducing plastic use and sharing turtle-protection messages.

## Lesson Development

### Introduction (8 min)

Ask learners: "Raise your hand if you've ever started a drawing, animation, or project and never finished it." *(Most hands will go up.)*

Explain: **The #1 problem in animation is not skill — it's never finishing.** Every animator has a folder of half-finished work. The difference between amateurs and professionals isn't talent — it's COMPLETION.

Today, we're going to plan and start a short animation about **Kasa the Green Turtle** — and we're going to learn the skills to actually FINISH it.

Show a photo of Kasa swimming in Watamu. Ask: "If we wanted to animate this turtle's story, how long do you think it would take?" *(Learners will guess wildly.)* "Today you'll learn why the answer is probably 2x whatever you just guessed."

### Step 1: Concept Introduction — Scoping & the Production Pipeline (15 min)

**Teach the Scope Test (5 min):**

Before starting ANY project, ask these 4 questions:

1. **Have I ever finished a project this big before?** → No? Make it HALF the size.
2. **How long did my last project take?** → Double that estimate.
3. **What's the simplest version of this idea?** → Make that first, add complexity later.
4. **Can I finish this in the time available?** → If not, cut scope, not quality.

**Scope progression** (write on board):

| Experience Level | Project Size |
|------------------|-------------|
| First animation | 3-5 seconds (bouncing ball) |
| After 5 animations | 10-15 seconds (walk cycle + action) |
| After 15 animations | 30-45 seconds (multi-shot scene) |
| After 30+ animations | 1-3 minutes (short film) |

**The 2x Rule:** Whatever time you THINK it will take, DOUBLE it. Animation always takes longer because you redo things, rendering takes time, and "quick fixes" snowball.

**Teach the 3-Phase Production Pipeline (10 min):**

| Phase | What Happens | % of Time | Output |
|-------|-------------|-----------|--------|
| **1. Pre-Production** (Plan it) | Write idea, storyboard, plan shots, gather reference, create assets | 20-30% | Storyboard + asset list |
| **2. Production** (Make it) | Block animation, review, add inbetweens, refine | 50-60% | Rough → refined animation |
| **3. Post-Production** (Finish it) | Clean up, color, sound, edit, render, review 3x | 20-30% | Finished animation! |

Show this planning template on the board:

```markdown
# Kasa Turtle Animation — Production Plan

## Project Idea (1 sentence):
A green turtle (kasa) glides through Watamu, sees a plastic bag, avoids it, swims on.

## Scope:
- Duration: 10 seconds
- Shots: 3 (establish, encounter, resolution)
- Experience: [first time / a few projects]

## Time Estimate (with 2x rule):
- My guess: 2 hours
- 2x reality: 4 hours
- Plan for: 4 hours (2 sessions of 2 hours)

## Phase 1: Pre-Production (1 hour)
- [ ] Write idea (done above)
- [ ] Storyboard 3 panels
- [ ] Gather turtle reference video
- [ ] Design kasa character (simple oval + flippers)

## Phase 2: Production (2 hours)
- [ ] Block shot 1: kasa gliding (rough poses)
- [ ] Block shot 2: kasa sees plastic bag, turns
- [ ] Block shot 3: kasa swims away safely
- [ ] Refine all shots (add inbetweens)

## Phase 3: Post-Production (1 hour)
- [ ] Color (green body, blue water)
- [ ] Sound (water sounds, gentle music)
- [ ] Export
- [ ] Watch 3 times, fix what bugs me

## Done Definition:
- [ ] All 3 shots animated (no missing frames)
- [ ] Colors filled (no white gaps)
- [ ] Exports without errors
- [ ] Watched 3 times — no major changes needed
- [ ] Someone else watched it and understood the story
```

### Step 2: Guided Practice — Scope & Storyboard Kasa's Animation (15 min)

Give each learner a **6-panel storyboard template** (paper). Walk through creating the plan together:

**1. Write the idea (2 min):**
"A green turtle (kasa) glides through Watamu Marine Park, sees a plastic bag floating like a jellyfish, turns away, and swims to safety."

**2. Apply the Scope Test (3 min):**
- Ask the class: "Has anyone here finished a 10-second animation before?"
- If no one has → scope to 5-8 seconds instead. **Cut scope, not quality.**
- Time estimate: "How long do you think this will take?" *(Learners guess.)* "Now double it. THAT is your real time budget."

**3. Storyboard 3-6 panels (10 min):**

Draw simple panels:
- **Panel 1 (Establishing, 3 sec):** Wide shot — kasa gliding through blue water with coral below. Slow, peaceful.
- **Panel 2 (Encounter, 2 sec):** Close-up — a plastic bag drifts into frame from the right. Kasa notices.
- **Panel 3 (Tension, 2 sec):** Kasa turns. The bag looks like a jellyfish. Will she eat it?
- **Panel 4 (Resolution, 3 sec):** Kasa turns away and swims upward to safety. The bag sinks out of frame.

**The Daily Check-In (2 minutes) — give each learner this card:**

```
┌─────────────────────────────────┐
│  DAILY CHECK-IN (2 minutes)     │
│                                 │
│  1. What did I do last time?    │
│  ___________________________    │
│                                 │
│  2. What am I doing today?      │
│  ___________________________    │
│                                 │
│  3. What's blocking me?         │
│  ___________________________    │
│                                 │
│  4. Am I on schedule?           │
│     [ ] Yes  [ ] No → cut scope │
└─────────────────────────────────┘
```

Explain: **Every work session, fill this out BEFORE you start animating.** It prevents the "I worked for 3 hours and have nothing to show" problem.

### Step 3: Independent Practice — Start Production (20 min)

Learners begin Phase 2 (Production) of their Kasa animation in Scratch or Krita.

**Challenge:** Animate **Shot 1 only** — Kasa gliding through the water (3-4 seconds). Use the storyboard panel 1 as reference.

**Scratch starter code for Kasa gliding:**

```scratch
when green flag clicked
set [x v] to (-200)
set [y v] to (0)
forever
  change x by (2)
  switch costume to [kasa-swim-1 v]
  wait (0.15) seconds
  switch costume to [kasa-swim-2 v]
  wait (0.15) seconds
  switch costume to [kasa-swim-3 v]
  wait (0.15) seconds
  switch costume to [kasa-swim-4 v]
  wait (0.15) seconds
  if <(x position) > [240]> then
    set [x v] to (-200)
  end
end
```

**Teacher circulates and checks:**
- Is the learner using their Daily Check-In card?
- Have they stuck to the scoped plan (just Shot 1)?
- Are they trying to make it "perfect" instead of "done"? → Remind them: **DONE IS BETTER THAN PERFECT.**

**Key teaching moment:** If a learner is stuck redoing the first frame over and over, intervene: "Block ALL the poses first — rough shapes only. Don't refine anything until every pose exists. Professionals block first, refine second."

### Step 4: Sharing & Feedback (7 min)

Learners do a **progress share** (not a finished-work share — this lesson is about process):

- Each learner shows their storyboard + whatever they've animated so far
- Class responds with:
  - "What's working?" (one thing)
  - "What's your next step?" (ask the creator — reinforces planning)

**Reinforce the Done Definition:** A project is DONE when:
- [ ] All shots are animated (no missing frames)
- [ ] Sound is synced (if using sound)
- [ ] Colors are filled (no white gaps)
- [ ] It exports without errors
- [ ] You've watched it 3 times and don't want to change anything major
- [ ] Someone else has watched it and understood it

It is NOT done when: "I just need to fix one more thing..." *(That's never true. There's always one more thing.)*

## Assessment

| Criteria | BE (Below Expectation) | AE (Approaching) | ME (Meets) | EE (Exceeds) |
|----------|------------------------|-------------------|------------|--------------|
| Applying the Scope Test | Does not evaluate project size; plans a project far too large | Identifies that project may be too big but cannot suggest a smaller version | Applies all 4 scope questions; plans a 10-15 sec animation with realistic scope | Applies scope test independently; justifies scope choice with evidence from past experience |
| Creating a production plan | No plan; starts animating immediately | Partial plan (storyboard only, no time estimate) | Complete 3-phase plan with storyboard, time estimate (2x rule), and done definition | Full plan + daily check-in used; tracks progress and adjusts scope when behind |
| Completing scoped work | Starts but produces no usable animation | Animates partial Shot 1 but no clear loop | Completes Shot 1 (kasa gliding) as a working animation | Completes Shot 1 + begins Shot 2; demonstrates "done > perfect" mindset |

## Extended Activity

**Homework / follow-up project:** Complete the full **Kasa's Journey** animation (all 3-4 shots) over the next 2 weeks using the production plan from this lesson. Bring the finished animation to class for a screening.

**Conservation connection:** Research one action you can take to help green sea turtles in Kenya. Options:
- Reduce single-use plastic (turtles mistake bags for jellyfish)
- Share a turtle conservation message on social media
- Learn about Local Ocean Conservation Trust in Watamu and what they do

Include a **conservation title card** at the end of your animation: "Green sea turtles (kasa) are endangered. Plastic pollution is a major threat. Reduce, reuse, recycle." Add this to your Done Definition checklist.

**Next lesson connection:** Lesson 47 covers **AI ethics** — when is it OK to use AI tools to help with your animation, and when does it hurt your learning?
